HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title defines the title of the webpage and is useful in SEO or Search Engine Optimization.
  • The style defines the css style for the webpage.
  • The Link command connects external and internal webpages to the page.
  • The meta command defines metadata such as keywords, author, description, and character set.
  • The base command defines the base url for the website.
  • The list types are ul for unordered list and ol for ordered lists.
  • The img command imbeds an image into the webpage. Using the src or source to direct to it's location.
  • The script command is used to imbed scripts.
  • The footer command is used to put authors, and copyright info at the bottom of a page.

CSS

  • Inline css code is created within the header html code to define the entire web page.
  • Internal CSS code is created in the header or body of the webpage html code.
  • External CSS style sheets can be linked within the header to make more specific style choices.
  • Separation of Concerns is when you separate code based on it's specific use.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A Selector is the element or attribute that the Declarations (or rules) will apply to.
  • A Property is what we want to apply to the Selector, and the Value for that property.
  • For example Display: Block causes an element that is displayed to start on a new line.
  • A CSS Box Model is a model to show how margins, borders, padding, and content are all nested inside each-other.
  • A * sets the style changes to all elements on the page. You can use specific terms such as header or footer to apply them to those elements individually.
  • A Class Attribute is a variable created in css that can be applied to any specified part of the html document.
  • Putting a . before a word such as .card creates a class by that name.

Git

  • git status: Checks what branch we are currently in.
  • git checkout - b branch-name: creates a new branch with that name and switches to it.
  • git add -a: save to commit (A stands for all)
  • git commit -m: add a commit message
  • git pull origin main: pull git repository from main branch. (should do before pushing your code)
  • git push origin feature/starter-code: push your code to be reviewed before being added to the main branch.
  • git checkout main: move to the main branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A const or constant variable has one unchanging definition or value.
  • Using let instead of const allows the variable to change value in the duration of runtime.
  • Falsy is if a value is 0, -0, On, "", null, undefined, or NaN. Otherwise it is Truthy.
  • Any value defined by === absolutely has to match the designated value or criteria to be true.
  • An array is a group of related data.
  • A boolean is a true or false value.
  • A string is a string of words that is surrounded by quotation marks.
  • An index is a number that marks a single piece of data's location within an array starting with 0.
  • variable.length defines the duration will be the length of the variable.
  • Commenting out code by using /* at the start and */ at the end keeps the code from being utelized.
  • A function is a set of instructions that tells the computer how to do a task.
  • You must first define a function and then call the function in order for it to execute.
